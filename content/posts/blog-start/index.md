## 블로그를 시작한 이유

ChatGPT 이후 블로그 트래픽이 줄었지만, 그래도 시작했다.

**보는 사람이 많이 없어도 나 자신을 정리하기 좋다.** 일기를 쓸 건 아니지만 글을 보고 쓰는 시간이 상당히 줄었다. 정기적으로 이런 시간을 갖는다는 건 내가 아는 것들을 다시 정리하는 시간이 될 것 같다.

**내가 소비한 시간을 컨텐츠로 만드는 시작점이라 생각했다.** 예를 들면 이 블로그를 만들기 위해 조사하며 소비한 시간을 컨텐츠로 만들 수 있다. 여러 컨텐츠로 만들 수 있을텐데, 블로그는 가장 간편하게 시작할 수 있다고 생각한다.

**만들어진 컨텐츠는 재생산이 가능하다.** 한 번 쓴 글을 인스타그램 포스트, 유튜브 쇼츠로 재가공할 수 있다. AI가 판을 치지만, 아직 유효한 전략이라 생각한다.

## 플랫폼 선택

선택지는 여러가지가 있었다.

- **네이버**: 유입에 유리한 환경이지만, 광고 수익보단 제휴 수익을 노려야 했다.
- **티스토리**: 네이버보다 덜해도 유입에 유리하지만, 마찬가지로 광고 수익보단 제휴를 노려야한다.
- **셀프호스트**: 플랫폼 제약이 없고 광고 대한 수익이 가장 많지만, 유입을 만들기 어렵다.

결국 셀프호스트를 선택했다.

어필리에이트나 광고 지면에 대해 제약받는 것도 싫고, 소유도 하고 싶었다.  
그리고 난 개발자이기에 셀프호스트 환경을 만드는데 어렵지 않을 거라 생각했다.  
(생각보다 번거롭긴했다.)

|                   | 네이버 블로그                   | 티스토리                                                                                       | 셀프호스트                     |
| ----------------- | ------------------------------- | ---------------------------------------------------------------------------------------------- | ------------------------------ |
| **광고 수익**     | 애드포스트만 가능 (낮은 수익률) | 애드센스 사용 가능. 단, 상단 광고는 카카오 수익 (2025년 광고 제한으로 블로거 수익 18-26% 감소) | 모든 광고 지면 통제, 수익 100% |
| **초기 설정**     | 간단                            | 간단                                                                                           | 복잡 (기술 필요)               |
| **플랫폼 종속성** | 높음                            | 높음                                                                                           | 없음 (독립 도메인)             |
| **초기 트래픽**   | 빠름 (네이버 검색 유리)         | 보통                                                                                           | 느림 (SEO 필요)                |

## 셀프호스트 선택: WordPress → Ghost → Hugo

셀프호스트를 알아보니 꽤 여러가지가 있었다.  
그 중 WordPress, Ghost, Hugo를 순서대로 써봤다.

### WordPress (1일 사용)

**문제점**:

- PageSpeed Insights 모바일 점수 50~60점 (검색노출에 불리)
- PHP, 플러그인 업데이트 등 유지보수 부담

WordPress는 사용하는 사람 수가 장점으로 보였다.  
힙스터 끼가 있는 나에겐 너무 꺼려지는 기술 스택이었다.

### Ghost (1주일 사용)

**문제점**:

- 1GB RAM으로 부족, 2GB 필요 (디지털 오션 월 $12)
- 뉴스레터를 사용하려면 Mailgun 구독 필요 (돈이 추가로 나감)
- 회원가입, 유료 구독 등 불필요한 기능

Ghost는 뉴스레터 중심 미디어나 유료 구독 모델에 적합해 보였다.  
단순 블로그에는 맞지 않았다. 기능이 많은 만큼 설치도 어렵다.

### Hugo (현재 사용 중)

정적 사이트 생성기다. 블로그 글을 파일로 만들어서 올리는 아주 단순한 구조다.

장점은 속도도 빠르고, 비교적 설정하기 쉽다. 비용도 1년 12달러라는 것(도메인 비용).  
테마도 꽤 많은 종류를 지원한다. [이곳](https://themes.gohugo.io/)에서 확인할 수 있다.

단점은 개발자가 아니면 러닝커브가 있다.  
**하지만 claude code, cursor를 사용한다면 진입장벽이 매우 낮아진다.**  
요즘 같이 비개발자도 cursor를 사용하는 시대에 잘 맞는다고 생각한다.

**실제 사용 결과**:

- 빌드 속도: 50개 페이지 200ms
- PageSpeed 모바일 점수: 95점 이상
- 비용: 무료 (Cloudflare Pages), 도메인만 연 $12

**장점**:

- 빠른 속도
- 서버 관리 불필요 (Git 푸시만 하면 자동 배포)
- 낮은 비용
- 보안 걱정 없음

**단점**:

- 댓글, 검색 등 외부 서비스 연동 필요
- 관리자 페이지 없음 (마크다운으로 작성)
- 초기 학습 필요
